Main page

Variables

Variable Type Description
Debris Service Equivalent to game:GetService("Debris"), because the game variable is not available
input table Contains the currently held inputs. up/down/left/right are bools
action1 and action2 are numbers (0 = not held, = 1 is pressed, >/>= 1 is held)
plr table Contains all player values.

Enum

Sulfur has it's own Enum table instead of Roblox's, containing properties for various things. It is read only, and accessed like Enum.Property.SubProperty
Enum Description
DefaultPhysics Default player physics. Can be set to plr.curPhysics
PartScale It's 0.125, or 1/8 of a stud, applied typically to object velocities
MonitorType Used when spawning a Monitor. Subproperties: Rings, Invincibility, Shoes, Life
BalloonType Used when spawning a Balloon. Subproperties: Blue, Yellow, Red

Functions

Function Arguments Description
spawnPart Spawns a part (because Instance.new is not available)
spawnObject objname:string, pos:Vector3, args:table Spawns an object. Not all default Sulfur objects are available, but it can spawn any custom object.
args is a list of settings that get applied to the object, such as Type for Monitors and Balloons. (See Enum)
addObject objname:string, args:table Adds an object to the Engine that can then be spawned via spawnObject, or placed inside of a level if it's in EditorDefinitions
playSound soundname:string, volume:number Plays a sound
setMusic musicname:string Changes the current music. (NOTE: This overrides the loaded level's Music setting)
